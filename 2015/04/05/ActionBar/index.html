添加AppCompat库

  • 打开Module中的build.gradle文件，在dependencies中添加如下内容：

      compile 'com.android.support:appcompat-v7:21.0.3'

建立Menu的布局

  • 在res/menu文件夹下建立discuss_add.xml文件，写入如下内容：

      <menu xmlns:app="http://schemas.android.com/apk/res-auto"
  xmlns:android="http://schemas.android.com/apk/res/android"
  xmlns:tools="http://schemas.android.com/tools"
  tools:context="net.coding.program.project.detail.TopicAddActivity">
  <item
      android:id="@+id/action_add_discuss"
      android:icon="@drawable/ic_menu_edit"
      android:title="发表评论"
      app:showAsAction="always" />
  </menu>

  • 其中android:id为调用该控件的id，android:icon为该控件的图标，即上图中右侧的白色铅笔，android:title为该控件的标题，在屏幕空间不够时，未显示的控件会隐藏到下拉列表中，并且只显示标题，app:showAsAction为控件的显示样式，有五种属性，分别为：
    1.ifRoom：会优先以图标的形式显示，如果屏幕空间不够则会隐藏到下拉菜单中
    2.never：永远都不会以图标形式显示，只会显示在下拉菜单中，并且只显示标题
    3.always：无论空间是否足够，都会以图标的形式显示
    4.whitText：在ActionBar上会尽量显示标题，但是如果空间有限但图标有效，标题可能显示不全
    5.collapseActionView：声明了这个操作的视窗应该被折叠到一个按钮中，当用户选择这个按钮时，这个操作视窗展开。否则，这个操作视窗在默认的情况下是可见的，并且即便在用于不适用的时候，也要占据操作栏的有效空间

  • 在使用app:showAsAction时需添加以下命名空间：

      xmlns:app="http://schemas.android.com/apk/res-auto"

修改主题样式

  • 使用appcompat-v7的ActionBar需要主题继承自Theme.AppCompat

  • 我们在res/velues/styles.xml中添加如下内容：

      <style name="AppBaseTheme" parent="Theme.AppCompat.Light.DarkActionBar">
      <!-- ActionBar的背景色 -->
      <item name="colorPrimary">@color/action_bar_background_color</item>
      <!-- 状态栏的背景色 -->
      <item name=”colorPrimaryDark”>@color/status_bar_color</item>
      <!-- ActionBar上字体的前景色 -->
      <item name="android:textColorPrimary">@color/white</item>
      <!-- 整个窗口的背景色 -->
      <item name="android:windowBackground">@color/view_background</item>
      <!-- 窗口中字体的颜色 -->
      <item name="android:textColor">@color/text_black</item>
  </style>

  • 套用官网的一张图片辅助说明Using the Material Theme

  • 修改好styles.xml之后，还需要在AndroidManifest.xml中对应的activity下调用android:theme="@style/AppBaseTheme"，这里我就直接在application下调用了，也就是对整个程序生效，代码如下：

      <application
      android:allowBackup="true"
      android:icon="@drawable/ic_launcher"
      android:label="@string/app_name"
      android:theme="@style/AppBaseTheme">
      <activity
          android:name="com.chenggoi.MainActivity"
          android:label="@string/app_name">
          <intent-filter>
              <action android:name="android.intent.action.MAIN" />
              <category android:name="android.intent.category.LAUNCHER" />
          </intent-filter>
      </activity>

      <activity android:name=".AddDiscussActivity" />
  </application>

修改Java类

  • 工作到这里就差不多快要结束了，对于Activity则直接继承ActionBarActivity，然后在onCreate中获取ActionBar：
      private ActionBar actionBar;
  actionBar = getSupportActionBar();

  • 接下来重写如下两个方法即可：

      @Override
  public void onCreateOptionsMenu(Menu menu, MenuInflater inflater) {
      //建立ActionBar，绑定布局
      inflater.inflate(R.menu.discuss_add, menu);
      super.onCreateOptionsMenu(menu, inflater);
  }

  @Override
  public boolean onOptionsItemSelected(MenuItem item) {
      //响应ActionBar上控件的选择事件，如有多个控件则可根据item值判断
      Intent intent = new Intent(mContext, AddDiscussActivity.class);
      startActivity(intent);
      return true;
  }

  • 对于Fragment则需要在主Activity上继承ActionBarActivity，然后在相应的Fragment页面重写以上两个方法，并且还要在onActivityCreated方法中添加setHasOptionsMenu(true);，如：

      @Override
  public void onActivityCreated(Bundle savedInstanceState) {
      super.onActivityCreated(savedInstanceState);
      setHasOptionsMenu(true);
  }
